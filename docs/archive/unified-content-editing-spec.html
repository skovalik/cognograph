The Actual Bug

NoteNode.tsx and TaskNode.tsx use EditableText for content display.

EditableText just renders raw text — it doesn't use RichTextDisplay!

// NoteNode.tsx line ~85
<EditableText
  value={nodeData.content || ''}
  onChange={(newContent) => updateNode(id, { content: newContent })}
/>

Problem Statement

The current "Rich Text / Plain Text" toggle creates a broken user experience:

Rich Text Mode
• Sidebar: WYSIWYG editor works ✓
• Card: Shows raw HTML tags ❌
<p><strong>Company:</strong> [Name]</p>
Plain Text Mode
• Sidebar: Raw markdown visible
• Card: Raw markdown visible ❌
**Company:** [Name]

Neither mode gives users what they expect: formatted content displayed correctly in both the sidebar editor AND the canvas card.

Root Cause (Simplified)

Node cards use EditableText which just renders raw text. They should use RichTextDisplay which already exists and handles HTML/markdown rendering.

Solution: Quick Fix with Double-Click Edit

The Fix

Replace EditableText with RichTextDisplay + wire double-click to open Properties Panel.

  • • Cards render HTML content properly (formatted)
  • • Double-click opens floating Properties Panel for editing
  • • Single-click still selects the node
  • • Toggle remains functional for users who want to see/edit source
~45 minutes to implement. Low risk. Preserves edit-on-double-click UX.
Double-Click Behavior
Before Opens inline textarea (raw text editing)
After Opens floating Properties Panel with full WYSIWYG editor

5. Double-Click Implementation Details

The double-click handler opens the floating Properties Panel:

// Store action
openFloatingProperties: (nodeId: string) => void

// Usage in component
const handleContentDoubleClick = useCallback((e: React.MouseEvent) => {
  e.stopPropagation()  // Prevent node selection change
  openFloatingProperties(id)
}, [id, openFloatingProperties])
Why Floating Properties?
  • • Works regardless of user's propertiesDisplayMode setting
  • • Opens directly on the content, feels like "editing in place"
  • • Contains full RichTextEditor with toolbar
  • • Can be pinned for multi-node editing
Alternative: Could check workspacePreferences.propertiesDisplayMode and call openProperties() for sidebar mode. But floating modal provides more consistent UX.

Remaining Edge Cases to Watch
  • Link clicks: <a href> in HTML content may navigate. Consider adding target="_blank" or event interception.
  • Text selection: Users may try to select text in rendered HTML. Could conflict with node drag behavior.
  • Cursor hint: Consider adding cursor: pointer or cursor: text to hint at editability.

6. Existing RichTextDisplay Code

For reference, here's what RichTextDisplay.tsx already does:

// RichTextDisplay already handles all these cases:

function isHtmlContent(content: string): boolean {
  return /<[a-z][\s\S]*>/i.test(content);
}

function markdownToHtml(text: string): string {
  return text
    .replace(/\*\*(.+?)\*\*/g, '<strong>$1</strong>')
    .replace(/__(.+?)__/g, '<strong>$1</strong>')
    .replace(/~~(.+?)~~/g, '<s>$1</s>')
    .replace(/`([^`]+)`/g, '<code>$1</code>')
    .replace(/\*([^*]+)\*/g, '<em>$1</em>')
    .replace(/_([^_]+)_/g, '<em>$1</em>')
    .replace(/\n/g, '<br>');
}

// Content processing logic:
const { htmlContent, isEmpty } = useMemo(() => {
  if (!content || content.trim() === '' || content === '<p></p>') {
    return { htmlContent: '', isEmpty: true };
  }

  // If content is HTML, use it directly
  if (isHtmlContent(content)) {
    return { htmlContent: content, isEmpty: stripHtml(content).trim() === '' };
  }

  // Plain text - check for markdown and convert
  if (hasMarkdownFormatting(content)) {
    return { htmlContent: markdownToHtml(content), isEmpty: false };
  }

  // Plain text - just convert line breaks
  return { htmlContent: content.replace(/\n/g, '<br>'), isEmpty: false };
}, [content]);

// Render with dangerouslySetInnerHTML
return (
  <div
    className={`${className} rich-text-display`}
    style={{ overflow: 'hidden', flex: 1, minHeight: 0, lineHeight: '1.5' }}
    onClick={onClick}
    onDoubleClick={onDoubleClick}
    dangerouslySetInnerHTML={{ __html: htmlContent }}
  />
);
Key Point: This component already handles HTML, markdown, and plain text. It's just not being used in the node cards!
